Detailed Description

Implements an inorder, pipelined, two FU (one for branch and the other for ALU) processor. This class provides the necessary methods to interface with the processor to successfully simulate a list of instructions

boolean org.binghamton.comparch.systems.Processor.cycle ( int  cycles)

Simulates the processor for the specified amount of clock cycles. If a HALT instruction is encountered, the simulation end when it executes the writeback stage and this method will return true. If a HALT instruction is not encountered, the method will return false once all the cycles have been executed.

Parameters
cyclesthe amount of clock cycles the simulation will run for
Returns
true if the processor was halted during the simulation; false otherwise
void org.binghamton.comparch.systems.Processor.initialize ( List< Instruction >  instructions)

Loads the list of instructions into program memory, sets the program counter to the beginning of program memory, clears out the pipeline, sets all general purpose registers and special register X to 0, and clears out data memory. This method should be called right after constructing an instance of a processor before any clockCycles are simulated. This method can also be used to rest the processor to the beginning state.

Parameters
instructionsa list of instructions, in program order, that will be loaded into instruction memory and simulated on the processor
